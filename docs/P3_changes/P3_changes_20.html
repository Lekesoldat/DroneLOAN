Navigation bar and footer

In P2 we stated that the navigation bar should be in a fixed position, however that is not the case for the final result. Our pages are not too long, so the users does not have to scroll that much causing it a problem not having the navigation stuck to the top of the screen. Also, the general page layout looked better without it being fixed at the top.

The width of the content in the navigation bar is also changed a bit from P2. We concluded that it looked better having all the page content at the same width.

Regarding the 1-click-policy, we followed it to a certain extent. You are able to get information about everything in one click, but for in-depth information and rental, two clicks are required.

Our 'curren-tab' does not have the bold text specified and shown in the mockup in P2. We made a grey background instead as it was easier to distinguish which tab actually was active.

Layout and Appearance

We changed from 'Courier New' to 'Trebuchet MS' because we felt 'Trebuchet MS' was more in line with the overall design of the site.

In P2 we specified thay each and every H1-tag would be 48px. We had to make exceptions to this rule because a 48px big H1 did not make sense design-wise on every page.

Content

Landing Page

When creating the top rented drones it seemed more natural for new users to have the quick ability to learn more about the drone rather than having the functionality of renting it immediately. Besides, the drone specific page was designed in such a way that when one scrolls to the bottom the page for renting the drone is at the bottom. So the functionality of renting the drone through the button is remained although the direct purpose is changed.

It also felt naturally to add a few keypoint elements underneath each drone so that the unexperienced user knows what he is looking at.

Initially we wanted the video to go into full-screen when scrolling underneath a certain point at the landing page. However, due to a lack of time we did not have time to implement it.

Modularity

There is not a single html file rendering the different pages. The different pages have their own .html file, except for the dronespecific-pages for each drone. Magnus created this page so that its content is rendered based on which drone was clicked on the 'our drones' page. It does so based on parameters passed in the URL.

The reason being for not implementing the 'load content' to 'main-section' feature was simply because it was a bit overkill for this project, and each member wanted to work on their own html-files respectively.

The navigation bar and footer renders itself when entering a page, and based on provided URL-parameters the 'currentTab' is displayed correctly.

Plan

New updated file structure

We followed the file structure planned in P2 to a certain extent. It turned out it wasn't necessary to have an individual JavaScript file for each page. Also, to create a self generating page and footer/navigation bar, more JavaScript files were needed. It has to be mentioned that all the JavaScript could be put in one file, but that would just be a horrible read and bad practice. Modules (ES6) was used and the outcome turned out to be rather clean.

Instead of having the videos saved locally, we uploaded them to YouTube and embedded them on the page. This was to reduce the page's load speed and provide better accessibility for the users as they can save them to their own playlists on YouTube if wanted.
